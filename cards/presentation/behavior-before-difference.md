## 洞察

数据分析报告的叙事顺序应该是：**先行为，再差异**。

1. **行为画像**：用户是谁 → 在做什么 → 满意度如何
2. **差异分析**：平台差异 → 嵌入差异 → 竞品差异

不要一上来就讲差异——听众还不知道"正常情况是什么样"，差异就没有锚点。

## 为什么

- 先画像建立"基准认知"，听众知道了正常情况后，差异才有冲击力
- 行为数据自然过渡到差异："PC 用户写邮件，Mobile 用户问健康"→ 那不同平台留存靠什么？
- 避免上来就甩一堆对比表，听众不知道看什么

## 具体案例

Edge Mobile Copilot Truth 第二幕：
- ❌ 之前：Feature 留存 → 留存话题 → Surface 全景 → 竞品 → UXR → Memory（跳来跳去）
- ✅ 之后：Persona → Intent → Intent TDR → Topic → Topic TDR → Feature 留存 → Editing 拆解 → Health 拆解 → Surface → 竞品 → UXR（先描述再分析）
