# Weekly Report 生成原则

## 结构
固定 **2 Edge Mobile 设计项目 + 1 Agent 项目**，上周完成 3 + 本周计划 3。

## 内容原则
1. **从设计师视角** — 说设计 effort 和设计决策，不是罗列消息
2. **按项目组织** — 项目来源是 C2 Design Plan 当前 sprint 的任务
3. **说自己做了什么** — 不总结别人的讨论
4. **简洁** — 每个项目一句话
5. **上周完成要有实质** — "方案进入 flight"、"Council 通过了什么"，而不是"report card 已提交"
6. **本周计划要有依据** — 来自用户口述或日历/项目文件，不能编

## 数据来源优先级
1. C2 Design Plan — 确定 sprint 和对应项目
2. 项目文件 (`projects/edge-mobile/*.md`) — 实际设计进展
3. 会议转录 (WorkIQ) — 补充关键决策
4. Daily logs — 辅助验证时间线，不作为主要来源
5. 用户口述 — 本周计划优先问用户

## 反模式
- ❌ 从 Teams 消息随意抓内容凑数
- ❌ 总结别人做的事当成自己的
- ❌ 把行政动作当完成内容
- ❌ 本周计划凭空编造
