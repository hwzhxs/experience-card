---
title: Memory entries stay short, details go to files
created: 2026-04-09
source: retro from CLAUDE.md rules
---

Every memory.db entry should be under 300 characters. If an insight needs more space, write the details to a project file (COOKBOOK.md, context.md, etc.) and store a pointer in memory: "→ see COOKBOOK.md §X". This keeps memory search fast and avoids context bloat on recall.
